Go constants are special variables with fixed values that remain unchanged throughout your program's execution. They offer a powerful way to improve code readability, prevent accidental errors, and make your codebase easier to maintain.

 

In this new guide of DevWithGo, we'll dive into how to define and use constants in Go, exploring best practices, different declaration methods, advanced techniques like grouping constants, and the intriguing concept of the iota identifier.

 

Go Constants

Go supports constant variables.

 

In Golang, constants have fixed values that do not change during the execution of a program.

 

It is possible to declare a constant in Go using the symbol const.

main.go

package main

func main() {
    const hoursInDay int = 24
}

Go support type and untyped constants, it is possible to give a data type or omit it. If you omit the data type, Go compiler will infer the data type by the value of the constant variable.

main.go

package main

func main() {
    const hoursInDay int = 24 // data type int
    const minutesInHour = 60 // data type int
}

As a var variable, const variable declarations are allowed at the package level, meaning outside of any functions. This makes the variables accessible throughout the entire package.

main.go

package main

import "fmt"

const projectName string = "DevWithGo.dev"

func main() {
    fmt.Println(projectName)
}

 

terminal

$ go run main.go
DevWithGo.dev

 

Go group of constants and iota

Go provides a concise syntax for declaring multiple constants together using a block enclosed in parentheses. This allows you to group related constants for better organisation and readability in your code.

main.go

package main

func main() {
    const (
        passwordMaxLength = 20
        loginMaxFailedAttempts = 5
    )
}

Declaring constants within a block is particularly useful when the constants are related and their values depend on each other. Within a constant block, you can use previously declared constants in the expressions of subsequent constants. Note that the order of declaration matters, as a constant can only reference constants that have already been declared within the same block.

main.go

package main

import "fmt"

func main() {
    const (
        hoursPerDay = 24
        minutesPerDay = hoursPerDay * 60
        secondsPerDay = minutesPerDay * 60
    )

    fmt.Println("Hours per day:", hoursPerDay)
    fmt.Println("Minutes per day:", minutesPerDay)
    fmt.Println("Seconds per day:", secondsPerDay)
}

 

terminal

$ go run main.go
Hours per day: 24
Minutes per day: 1440
Seconds per day: 86400

 

When working with constants in Go, it's important to understand the concept of iota. Iota is a special identifier that simplifies the declaration of auto-incrementing integer constants within a const block.

main.go

package main

import "fmt"

func main() {
    const (
        zero = iota
        one
        two
        three
    )

    fmt.Println(zero)
    fmt.Println(one)
    fmt.Println(two)
    fmt.Println(three)
}

 

terminal

$ go run main.go
0
1
2
3

 

Why use constants in Go?

Constants are a fundamental concept in programming, and Go is no exception. They offer several key benefits that can significantly improve the quality of your code.

There are some benefits of using constants variable in Golang, like:

  • Performance: constants are determined at compile time, not at runtime, this can improve the performance
  • Readability: values of the constants never change, this improve the readability of your code
  • Maintainability: when you need to update a value that's used in multiple places, you only have to change it in the constant declaration, making maintenance a breeze
  • Error Prevention: the immutability of constants prevents accidental modifications, helping you avoid introducing errors into your code
