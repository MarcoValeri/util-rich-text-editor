Go if and else statement allows you to master conditional logic. In this guide, we'll unravel the power of if, else if, and else statements in Golang.

You will learn how to create decision-making structures in your programs, allowing different blocks of code to execute based on specific conditions.

Discover best practices for writing clean, efficient, and effective conditional logic in your Go applications.

 

Go if and else statement

In Go, conditional statements allow you to control the flow of your program based on whether certain conditions are met. This enables your code to make decisions, execute specific blocks of code only when necessary, and adapt to different scenarios.

 

main.go

package main

import "fmt"

func main() {
    if true {
        fmt.Println("TRUE")
    }
}

 

terminal

$ go run main.go
TRUE

 

Unlike languages like JavaScript, where parentheses are often required around conditions in if statements, Go does not mandate their use. As shown in the previous example, the condition can be directly written after the if keyword without parentheses.

 

Go if statement

In Go, an if statement allows you to execute a block of code only if a specified condition evaluates to true.

 

main.go

package main

import "fmt"

func main() {
    programmingLanguage := "Go"
    if programmingLanguage == "Go" {
        fmt.Println("Hello", programmingLanguage)
    }
}

 

terminal

$ go run main.go
Hello Go

 

Go if else statement

In Go, an if statement (optionally followed by an else statement) allows you to conditionally execute different blocks of code based on whether a specific condition evaluates to true or false.

 

main.go

package main

import "fmt"

func main() {
    isAdminUser := true
    if isAdminUser {
        fmt.Println("Show admin dashboard")
    } else {
        fmt.Println("Do not show admin dashboard")
    }
}

 

terminal

$ go run main.go
Show admin dashboard

 

You can use an if statement in Go without a corresponding else clause. However, an else clause cannot exist on its own; it must always be paired with a preceding if statement.

 

main.go

package main

import "fmt"

func main() {
    isAdminUser := true

    // if statement without else
    if isAdminUser {
        fmt.Println("Show admin dashboard")
    }

    // if statement with else
    if isAdminUser {
        fmt.Println("Show admin dashboard")
    } else {
        fmt.Println("Do not show admin dashboard")
    }
}

 

Go if, if else and else statement

In Go, you can combine if, else if, and else statements to create complex decision-making logic.

 

main.go

package main

import "fmt"

func main() {
    providedNumber := 0
    if providedNumber > 0 {
        fmt.Println(providedNumber, "is a positive integer")
    } else if providedNumber < 0 {
        fmt.Println(providedNumber, "is a negative integer")
    } else {
        fmt.Println(providedNumber, "is neither positive nor negative")
    }
}

 

terminal

$ go run main.go
0 is neither positive nor negative

 

As you can see, in the above example the variable providedNumber is an integer equal to zero. This means that 0 is neither positive nor negative.

 

In Go, you can declare and initialise variables directly within the condition of an if statement (or its else if and else clauses). This allows you to define variables with a scope limited to the conditional block, simplifying your code and enhancing readability.

 

main.go

package main

import "fmt"

func main() {
    if providedNumber := -1; providedNumber > 0 {
        fmt.Println(providedNumber, "is a positive integer")
    } else if providedNumber < 0 {
        fmt.Println(providedNumber, "is a negative integer")
    } else {
        fmt.Println(providedNumber, "is neither positive nor negative")
    }
}

 

terminal

$ go run main.go
-1 is a negative integer

 

Go nested if else statement

In Go, you can nest conditional statements, meaning you can place an if, else if, or else statement inside another if, else if, or else block. This technique allows you to create more complex decision-making structures where the execution of one code block depends on the outcome of multiple conditions.

 

main.go

package main

import "fmt"

func main() {
    authentication  := true
    userRole  := "admin"
    if authentication {
        if userRole == "admin" {
            fmt.Println("Login and redirect to admin dashboard")
        } else {
            fmt.Println("Login and redirect to basic dashboard")
        }
    }
}

 

terminal

$ go run main.go
Login and redirect to admin dashboard

 

The ternary operator in Go

Unlike some other programming languages like JavaScript, Go does not include a built-in ternary operator. The ternary operator is a concise way to write conditional expressions in a single line.

 

main.go

package main

import "fmt"

func main() {
    authentication := true
    authentication ? fmt.Println("TRUE") : fmt.Println("FALSE") // this is not allowed, the compiler will return an error
}

 

terminal

$ go run main.go
# command-line-arguments
.\main.go:7:20: invalid character U+003F '?'
.\main.go:7:22: syntax error: unexpected fmt at end of statement

 

Why does Go not use the ternary operator? Golang intentionally omits the ternary operator due to concerns about code readability and maintainability. While the ternary operator can be concise, it can also lead to complex and less intuitive expressions when used excessively or for intricate conditions.

The Go language prioritises explicitness and clarity, favouring the traditional if-else statement structure, even if it requires a few extra lines of code. This approach promotes code that is easier to understand, debug, and maintain, aligning with Go's philosophy of simplicity.

 

main.go

package main

import "fmt"

func main() {
    authentication  := true
    if authentication {
        fmt.Println("TRUE")
    } else {
        fmt.Println("FALSE")
    }
}

 

terminal

$ go run main.go
TRUE
