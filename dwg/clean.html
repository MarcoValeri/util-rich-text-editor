In programming languages, comments play a crucial role in describing sections of code or leaving instructions for fellow developers.

While well-structured code with clear variable and function names can often reduce the need for excessive comments, they remain valuable for providing context, explanations, or insights that may not be immediately obvious from the code itself.

Let's delve into how Go, a popular programming language, handles comments.

 

Go comments

Go offers two distinct ways to incorporate comments into your code:

  • Single-line Comments: these begin with two forward slashes (//) and continue until the end of the line. They are ideal for short notes or explanations directly beside the relevant code
  • Multiline Comments: these start with a forward slash and asterisk (/*) and end with the reverse (*/). They are perfect for longer explanations, documentation blocks, or temporarily disabling sections of code

 

main.go

package main

import "fmt"

func main() {
    // Declare a variable representing a website URL
    website := "www.devwithgo.dev"

    /**
    Print the website URL to the console
    using the fmt.Println() function
    */
    fmt.Println(website)
}

 

terminal

$ go run main.go
www.devwithgo.dev

 

In programming languages, comments serve a purely informational purpose and are not involved in the program's logic. They are intended for human readers, providing explanations, documentation, or notes within the code. As demonstrated in the previous Go example, the compiler (or interpreter) ignores comments entirely during the process of executing the program.

 

Go single-line comments

In Go, single-line comments are created using two forward slashes (//).

Any text following the // on the same line is considered a comment and is ignored by the Go compiler.

This makes single-line comments ideal for brief explanations or annotations directly beside the corresponding code.

 

main.go

package main

import "fmt"

func main() {
    // Store user email address
    userEmail := "user@gmail.com"

    // Store user password (for illustrative purposes only, not secure)
    userPassword := "1234"

    // Print out user email and password
    fmt.Println("Your login data")
    fmt.Printf("email: %s  - password: %s", userEmail, userPassword)
}

 

terminal

$ go run main.go
Your login data
email: user@gmail.com - password: 1234

 

Go multiline comments

In Go, multiline comments are enclosed between the delimiters /* and */.

These comments can span multiple lines, making them ideal for longer explanations, documenting code blocks, or temporarily disabling sections of code during development.

 

A common convention within multiline comments is to use the TODO tag to mark areas where work still needs to be done or improvements can be made.

This practice helps developers track outstanding tasks and easily identify areas for future attention.

 

main.go

package main

func main() {
    productId := 1
    productTitle := "Go Book"
    productPrice := 9.99

    /*
    TODO:
    print out the above
    product data with
    fmt.Printf() functions
    */
}

 

While /* ... */ is the designated syntax for multiline comments in Go, you can achieve a similar effect using multiple consecutive single-line comments (//).

This technique is often employed for shorter multiline comments, where the visual separation of individual lines can enhance readability.

Ultimately, the choice between the two methods often comes down to personal preference or project conventions.

 

main.go

package main

import "fmt"

func main() {
    // Store product data in separate variables
    // for product ID, title, and price
    productId := 1
    productTitle := "Go Book"
    productPrice := 9.99
    
    fmt.Println("Product data")
    fmt.Printf("Product ID: %d\n", productId)
    fmt.Printf("Product Title: %s\n", productTitle)
    fmt.Printf("Product Price: %g\n", productPrice)
}

 

terminal

$ go run main.go
Product data
Product ID: 1
Product Title: Go book
Product Price: 9.99

 

Using comments for testing in Go

Comments are frequently used as a tool for testing and debugging code.

Developers often temporarily comment out sections of code to isolate issues, experiment with potential solutions, or simply gain a better understanding of how different parts of the program interact.

This practice is widespread across programming languages, including Go.

 

main.go

package main

import "fmt"

func main() {
    productId := 1
    productTitle := "Go Book"
    productPrice := 9.99
    productAvailable := true
    
    fmt.Println("Product data")
    fmt.Printf("Product ID: %d\n", productId)
    fmt.Printf("Product Title: %s\n", productTitle)
    fmt.Printf("Product Price: %g\n", productPrice)
}

 

terminal

$ go run main.go
# command-line-arguments
.\main.go:9:2: productAvailable declared and not used

 

As you can see, the above code creates an error because the variable productAvailable is declared but not used – Go does not allow this.

Let's add a comment to temporarily disable this line of code for testing purposes.

 

main.go

package main

import "fmt"

func main() {
    productId := 1
    productTitle := "Go Book"
    productPrice := 9.99
    // productAvailable := true
    
    fmt.Println("Product data")
    fmt.Printf("Product ID: %d\n", productId)
    fmt.Printf("Product Title: %s\n", productTitle)
    fmt.Printf("Product Price: %g\n", productPrice)
}

 

terminal

$ go run main.go
Product data
Product ID: 1
Product Title: Go book
Product Price: 9.99

 

With the commented-out line, the program now compiles and runs without errors. This simple example demonstrates how comments can be a valuable tool for testing and debugging Go code.
