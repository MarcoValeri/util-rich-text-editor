In this new tutorial of DevWithGo, I will show you how to create a form with a rich text editor field using Go.

Integrating a WYSIWYG editor in your Golang application is an important and useful task. It allows you to improve the quality of your products by providing a user-friendly way to format text.

 

I am going to work with Go, JavaScript and Quill, a modern rich text editor, an open source WYSIWYG created by Jason Chen, Byron Milligan and actively maintained by Slab - you can find more information on Quill Rich Text Editor on GitHub.

 

This tutorial is perfect for Go developers who want to enhance their web forms with a rich text editing experience.

If you want code with me in this tutorial, make sure you have Go installed and your favourite code editor ready - I am using Visual Studio Code.

 

Let’s start.

 

How to create a rich text editor field with Go and Quill

As a first step, we have to create a folder where we store our app.

I am going to call this folder go-rich-text-editor-with-quill, feel free to call it as you prefer.

 

terminal

$ mkdir go-rich-text-editor-with-quill

 

Now, open the go-rich-text-editor-with-quill folder (or your chosen folder name) in Visual Studio Code or your preferred code editor.

 

terminal

$ code go-rich-text-editor-with-quill

 

It’s time to create a main.go file in the root of our go-rich-text-editor-with-quill folder. Since this is a small project with a simple structure, we’ll keep all of our Go code within this single file.

Ready to get started with the implementation? Let’s write some code.

 

main.go

package main

import "fmt"

func main() {
    fmt.Println("Hello Go")
}

 

Let’s test out setup by running this simple code snippet, which prints “Hello Go” to our terminal. To do this, we first need to initialise our Go module using the go mod init command. This command sets up a module path for our project - you can learn more about Go modules in the official Go documentation.

 

terminal

$ go mod init gorichtexteditorwithquill
go: creating new go.mod: module gorichtexteditorwithquill
go: to add module requirements and sums:
        go mod tidy

 

With our Go module initialised, we're ready to run our code and see “Hello Go” printed in the terminal.

 

terminal

$ go run .
Hello Go

 

Now that we’ve confirmed our basic Go setup, let’s create the structure for our web application. We already have our main.go file; now we need an HTML file to hold our form and a JavaScript file to configure Quill and handle the form data.

 

You can create the following files and folders using your terminal, or through your code editor’s interface:

  • public/js/quill-editor.js
  • views/home.html

 

I am going to use the terminal again.

 

terminal

$ mkdir public
mkdir public/js
touch public/js/quill-editor.js
mkdir views
touch views/home.html

 

At this point, your project structure should be like the following.

 

main.go

go-rich-text-editor-with-quill/
└── public/
    └── js
        └── quill-editor.js
└── views/
    └── home.html
└── go.mod
└── main.go

 

Now, let’s create the home.html template file in the views folder. This file will house our form and incorporate the Quill library. As outlined in the Quill Quickstart documentation, we need to include both a stylesheet file and the Quill library itself to get started. Once we have added those, we will initialise the Quill editor and construct our form within the template.

 

views/home.html

DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">>
