In this new tutorial of DevWithGo, I will show you how to create a form with a rich text editor field using Go.

Integrating a WYSIWYG editor in your Golang application is an important and useful task. It allows you to improve the quality of your products by providing a user-friendly way to format text.

 

I am going to work with Go, JavaScript and Quill, a modern rich text editor, an open source WYSIWYG created by Jason Chen, Byron Milligan and actively maintained by Slab - you can find more information on Quill Rich Text Editor on GitHub.

 

This tutorial is perfect for Go developers who want to enhance their web forms with a rich text editing experience.

If you want code with me in this tutorial, make sure you have Go installed and your favourite code editor ready - I am using Visual Studio Code.

With our home.html template in place, let’s add some JavaScript code to the quill-editor.js file. This code will initialise the Quill library and handle the form submission, allowing us to retrieve the formatted text content when the user submits the form.

 

public/js/quill-editor.js

window.onload = () => {
    // Initialize Quill editor
    const quill = new Quill('#editor', {
        theme: 'snow'
    });

    // DOM variables
    const form = document.getElementById('form');
    const formContentInput = document.getElementById('form-content-input');

    /**
    * JS submit event that gets data from the form
    * and
    * pass quill content to the hidden input
    */
    form.addEventListener('submit', () => {
        const getContent = quill.root.innerHTML;
        formContentInput.value = getContent;
    })
}

 

In the above file, I have initialised the Quill library with const quill = new Quill('#editor').

Make sure that #editor ID here matches the corresponding <div id=”editor”></div> element’s in the home.html file - you can use a different name if you prefer it.

 

Inside the new Quill(‘#editor’ {}) I have just specified the theme: snow, but this beautiful library allows you to customise the WYSIWYG editor as you want - you can find more information in the official documentation, in the section Quill Configuration.

 

With our frontend structure in place, let’s now focus on the backend by creating a local server in our main.go file. This server will handle the form submission and process the data we receive from the rich text editor when the user submits their content.

 

main.go

package main

import (
    "fmt"
    "html/template"
    "net/http"
)

func home() {
    tmpl := template.Must(template.ParseFiles("./views/home.html"))
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {

        // Get data from the form
        getFormSubmit := r.FormValue("form-submit")
        getContent := r.FormValue("form-content")

        /**
        * Print out in the terminal
        * the form-content
        * if the form has been submitted
        */
        if getFormSubmit == "Send your content" {
            fmt.Println(getContent)
        }

        tmpl.Execute(w, nil)
    })
    }

func main() {
    // Set static files under public folder
    fs := http.FileServer(http.Dir("./public"))
    http.Handle("/public/", http.StripPrefix("/public/", fs))

    home()
    http.ListenAndServe(":80", nil)
    }

 

In the code above, the home() function is responsible for rendering the home.html template and handling form submissions, including extracting and processing data from the rich text editor. The main() function configures our server to serve static files from the public directory, ensuring that the Quill library is accessible to the browser. It then sets up the necessary rotes and starts the server on port :80.
