In Go, the switch statement offers a powerful alternative to if/else chains for handling multiple conditional branches.

While both structures can be used for decision-making, each excels in different scenarios.

Let's dive into the nuances of the switch statement and explore when it might be a more suitable choice than traditional if/else constructs in your Go code.

 

Go switch statement

Go also offers the switch statement, providing an alternative to if/else chains for handling multiple conditional branches.

The switch statement evaluates an expression and then checks for matches against several potential cases. If a match is found, the corresponding block of code is executed.

 

main.go

package main

import "fmt"

func main() {
    month := 1
    
    switch month {
    case 1:
        fmt.Println("January")
    case 2:
        fmt.Println("February")
    case 3:
        fmt.Println("March")
    case 4:
        fmt.Println("April")
    case 5:
        fmt.Println("May")
    case 6:
        fmt.Println("June")
    case 7:
        fmt.Println("July")
    case 8:
        fmt.Println("August")
    case 9:
        fmt.Println("September")
    case 10:
        fmt.Println("October")
    case 11:
        fmt.Println("November")
    case 12:
        fmt.Println("December")
    default:
        fmt.Println("Invalid value")
    }    
}

 

terminal

$ go run main.go
January

 

In the above example, the variable month is equal to 1, so the program executes the code associated with case 1, printing "January" as the output.

 

Go's switch statement also features a default case. The code block within the default case is executed if none of the other cases match the value being evaluated. This provides a fallback option when no other cases apply.

 

main.go

import "fmt"

    month := 15

In some programming languages like JavaScript, each case within a switch statement typically requires a break keyword to prevent "fallthrough", where execution continues into subsequent cases.

However, Go's switch statement behaves differently. By default, Go automatically prevents fallthrough, so you only need to use break if you explicitly want this behaviour.

 

Go switch statement with fallthrough

In Go's switch statement, the fallthrough keyword allows execution to continue into subsequent cases, even after a match is found.

This behaviour is distinct from languages like JavaScript, where break statements are typically needed to prevent fallthrough.

 

main.go

package main

import "fmt"

func main() {
    today := 3

    fmt.Println("Workdays remaining in this week:")
    switch today {
    case 1:
        fmt.Println("Monday")
        fallthrough
    case 2:
        fmt.Println("Tuesday")
        fallthrough
    case 3:
        fmt.Println("Wednesday")
        fallthrough
    case 4:
        fmt.Println("Thursday")
        fallthrough
    case 5:
        fmt.Println("Friday")
    default:
        fmt.Println("Invalid value")
    }    
}

 

terminal

$ go run main.go
Workdays remaining in this week:
Wednesday
Thursday
Friday

 

Go switch with multiple cases

In Go, a switch statement can execute a block of code based on whether a single value matches one or more specified cases.

 

main.go

package main

import "fmt"

func main() {
    today := "Monday"
    switch today {
    case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday":
        fmt.Println("Weekday")
    case "Saturday", "Sunday":
        fmt.Println("Weekend")
    default:
        fmt.Println("Invalid value")
    }   
}

 

terminal

$ go run main.go
Weekday

 

Go switch without expression

In Go, each case within a switch statement can have its own expression to evaluate, providing greater flexibility and control compared to traditional switch statements in other languages.

 

main.go

package main

import "fmt"

func main() {
    income := 80000
    switch {
    case income <= 12500:
        fmt.Println("Tax rate 0%")
    case income > 12500 && income <= 50000:
        fmt.Println("Tax rate 20%")
    case income > 50000 && income <= 125000:
        fmt.Println("Tax rate 40%")
    case income > 125000:
        fmt.Println("Tax rate 45%")
    default:
        fmt.Println("Invalid value")
    }   
}

 

terminal

$ go run main.go
Tax rate 40%

 

Go switch statement: optional expressions

In Go, you can initialise variables within a switch statement, even when omitting the expression to be evaluated.

This allows you to create variables with a scope limited to the switch block.

 

main.go

package main

import "fmt"

func main() {
    switch season := 1; season {
    case 1:
        fmt.Println("Winter")
    case 2:
        fmt.Println("Spring")
    case 3:
        fmt.Println("Summer")
    case 4:
        fmt.Println("Autumn")
    default:
        fmt.Println("Invalid value")
    }   
}

 

terminal

$ go run main.go
Winter

 

Switch vs if else statement

Go's switch and if/else statements both offer powerful ways to handle conditional logic. When faced with multiple conditions, the switch statement often shines due to its cleaner syntax and easier readability.

It excels at comparing a single value against several possibilities, presenting a more organised structure than nested if/else chains.

However, for complex conditions or scenarios where you need to combine multiple logical expressions, if/else might be the more flexible choice.

 

In terms of performance, the switch statement can sometimes be faster than an equivalent if/else chain, especially when dealing with a large number of cases.

This is because Go's compiler can often optimise switch statements into jump tables, allowing for efficient execution.

However, the performance difference is usually negligible for smaller cases or more complex conditions.

The primary factor in choosing between the two should be code clarity and maintainability.
